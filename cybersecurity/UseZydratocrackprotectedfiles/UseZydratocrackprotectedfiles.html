Crack a protected file with Zydra

In this tutorial I’m showcasing a tool called Zydra, and using it in order to crack a password protected file. Keep in mind that I’m going to be using a Linux computer in this article, so if you want to follow along, make sure to have any Linux distribution available. The technique used is just a simple BruteForcing.

Why would someone lock their files with a password?

Well, let’s say that someone manages to remotely (or physically) gain access to your computer. If someone has important files inside it, besides having access to the computer, without that password files will remain locked and safe. Or maybe not?

How to encrypt and protecta a file?

Every cybersecurity lover, in order to crack and exploit a system, has to at least know how the procedure of what you’re trying to hack works, so we are first going to encrypt a simple image.png file. We are going to create an encrypted .zip file containing the png by simply executing this command:

Zip -e password_protected.zip image.png

In this line -e stands for “encrypt”, password_protected.zip is just the name of the zip file we will create, and everything after that are the elements inside (in this case just our png). Now we can unzip the png, and it will stay protected.

Let's make sure we have Zydra on our computer

For those of who don’t have Zydra downloaded already, this is the git clone link:

git clone https://github.com/hamedA2/Zydra.git

Make sure to execute the line in the folder you want to store Zydra in.

Use Zydra to crack a file's password

Now that we have it downloaded, we can begin. First of all, let’s cd to the Zydra directory

cd Zydra/

Inside this directory we will find a Zydra.py file, which is the script we are going to use to brute force the file.
Every Linux distribution has Python already pre-installed, but make sure to execute the script with Python3 and not 2. Now I’m going to show you one of the methods, which is quite slow (works well for small passwords)

Python3 Zydra.py -f password_protected.zip -b letters,digits -m 1 -x 6

-f stands for “file”, and you want to insert the file you want to crack, -b is used to limit the char type (in this case, I tell Zydra to only try and use passwords with letters and digits). The -m is used to tell Zydra the minimum length of the passwords to try, and -x the maximum.
However, using this method will take too much time for long passwords and so usually we use a word list to make things faster. If you don’t have a word list, just google it and download one. For this tutorial I’m going to use the Kali pre-installed word list “rockyou.txt”.

Python3 Zydra.py -f password_protected.zip -d rockyou.txt

-d stands for dictionary, and you just write the name of the words list file.
This method is so much faster. Keep in mind that the success rate of it depends on the word list you are using. If you don’t get a match, try using another one and don’t get upset. People use simple passwords in order to remember them, and you will most likely find it in the most common words list.
