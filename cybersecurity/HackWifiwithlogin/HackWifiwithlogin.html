Hack an open Wi-Fi with login

Have you ever found yourself in an Hotel, Airplane, Coffee Shop, and suddenly you see an apparently open Wi-Fi. But when you actually try and connect to it, you will be redirected to an authentication page where you will be asked to login. With MAC spoofing it actually is possible to avoid this. Keep in mind that as in every cybersecurity tutorial, I am using Linux. MAC spoofing is possible for every operating spoofing, but if you want to follow along, then make sure to be on Linux.

But what is a MAC address?

MAC stands for Media Access Control address, and it is a unique identifier that every network interface controller (NIC) of any device has assigned for connecting to a network address. MAC addresses are made by six groups of two hexadecimal digits, and it is assigned by the device manufacture.

How are we going to proceed?

We are going to do something called MAC spoofing, which basically allows your device to pretend being another one by faking another MAC address of another device (as MAC addresses are unique for every device in the world). The reason why this works with Wi-Fis such as the ones in a Coffee Shop, is because they keep track of who is allowed and who’s not thanks to a whitelist. Once someone is authorized, it gets into the whitelist where it is identified via its MAC address.
What we need to do then is scan the area and find a device that is already authorized, spoof its address and pretend to be it.

Disclaimer:

Keep in mind that in order to do this we need a network card that can be put into wireless monitor mode. Some new laptops already have a decent enough card that can get into monitor mode, but as you most likely won’t have one, I’ll leave you the link of the one I use *here*, so you can have an idea of what that is, and eventually either purchase that one or another.

Make sure to have all the tools required

If you are using Kali you will most likely have all of them already preinstalled (macchanger, aircrack-ng), but for those of you who are not using it, I’ll go through the process in order to simply install them. First open your terminal and type

apt-get install macchanger aircrack-ng

Randomize our MAC address

Just to be more secure, we want to randomize our current MAC address as we are connecting to a public network. In order to do that open the terminal and type

sudo ip a

you will now see a list of all your network cards. Now just identify the name of the network card you are using (if you don’t know it, it most likely is “wlan0”). After doing that, we need to disable our network card, and then just randomize it by typing. In order to disable the card we type in the terminal

sudo ifconfig wlan0 down

In this line I considered “wlan0” being your network card, but just switch it with whatever your card’s name is. Now we can randomize the address with

sudo macchanger -r wlan0

after doing so make sure you have your network card up with

sudo ifconfig wlan0 up

Let's begin

First thing to do is to put your network card into monitor mode. If you don’t know how to that don’t worry, as it is a simple process:

sudo airmon-ng start wlan0

In the previews line I assumed that your network’s card name is “wlan0”. After putting it in monitor mode, the name would become “wlan0mon”.

Find an authenticated device

Now that the card is in monitor mode, we can start searching for other devices with

sudo airodump-ng wlan0mon

What you will see will be something like this:

What you are looking at is a list of all the operating networks. Once identified the network you can press Ctrl+C in order to stop refreshing (don’t worry, the list won’t disappear). The information you need to get from the network is the channel it is running in

Once that’s identified we can proceed with

Sudo airodump-ng wlan0mon -c 11 - -encrypt OPM

-c stands for channel, and the number after it is the channel number. OPM tells the program we only want to see open networks. Now we will have a list with all the devices that are connected, or waiting for a connection, on that network. The ones connected are our victims for MAC spoofing.

Disclaimer

Be careful with this as it could cause an IP collision, which consists in the router trying to give connection to us, and the victim at the same time. This might cause a DOS attack (Denial Of Service) and you can get in trouble for this

Final step: gain access

First of all make sure to have the victim’s MAC address either copied or written somewhere.
After doing that we need to take our network card down again

sudo ifconfig wlan0 down

Now let’s change the MAC address with

sudo macchanger -m EE:3C:4C:D3:GA:35 wlan0

The hexadecimal sequence you see in the line above is victim’s MAC. Now we can bring the card up again

sudo ifconfig wlan0 up

We now successfully gained access to the network!
